Prerequisites

To flash right now — that's all you need:
  • A laptop or desktop (not a phone or tablet).
  • Chrome 89+ or Microsoft Edge. ❌ Safari and Firefox can't flash — they don't support WebSerial.
  • An ESP32 board (the Chicken Tender brain).
  • A USB data cable that fits the board. ⚠️ Many cheap cables only charge — if the board never shows up, the cable is the #1 culprit.
  • No installs. No Arduino, no Python, no drivers to download by hand — the browser does it all.

🪟/🍎 One-time USB driver: a few boards use a CH340 or CP210x USB chip that Windows/macOS may need a small driver for. If your board never appears in the port list even with a good cable, install that driver once (search your board model + "CH340 driver" or "CP210x driver"), then try again.

📡 What you'll also need to actually USE the coop afterward

Flashing puts the program on the chip. To make the coop do things, the board joins your WiFi and talks to the Tender Cells backend on your network:

  • Your home WiFi name + password (you enter these on the board after flashing — Step 4). ⚠️ It must be a 2.4 GHz network — ESP32 boards can't see 5 GHz-only WiFi.
  • The Tender Cells backend running on a computer on the same network — start it with npm run demo from the repo. It prints a broker IP you'll type into the board.
  • That's it — no cloud account, no internet required. Everything stays on your LAN.

Don't have the backend yet? You can still flash now and set up WiFi later — the board just waits.

Step 1 — Pick what to flash

New here, or using a small dev board like a Seeed XIAO ESP32-S3? Start with Starter Node — it needs no coop hardware and proves everything works. One Starter Node binary works for any product (Chicken Tender, Roaming Roost, Duck Dock, WatchTower, …) — you pick which product it represents during WiFi setup (Step 4). The flasher checks your chip on connect and won't let you flash the wrong one.

Step 2 — Plug the board into this computer

Use the USB cable. A light on the board should turn on.

Step 3 — Press the button

  1. A little window pops up asking which USB port. Pick the one that appears when the board is plugged in (often named CP210x, CH340, or USB Serial), then press Connect.
  2. Press Install (it names the board you picked) and confirm. A green bar fills up.
  3. When it says done, the program is on the chip. Unplug or keep going to WiFi setup below. 🎉
❓ Board doesn't show up / install fails
  1. Swap the cable for a known data cable — #1 cause.
  2. Hold the BOOT button on the board, press the button above, keep holding BOOT until the green bar starts, then let go.
  3. Close any other program using the port (Arduino IDE Serial Monitor, etc.).
  4. Still stuck? Switch to Learn mode and use the Arduino IDE path.
❓ It flashed, but no setup WiFi network appears
  1. Unplug and replug the board. The flasher holds the USB port; a fresh power-up boots the new program, which then starts the setup network.
  2. Wait ~10 seconds, then re-scan WiFi on your phone for TenderNode-Setup (Starter Node) or ChickenTender-Setup (coop). It's an open network — no password.
  3. Watch what the board is doing over USB (see the Serial Monitor tip below) — the boot banner and [WIFI] Starting setup portal … line confirm it's alive.
🔎 See what the board is doing (Serial Monitor)

The board prints a live log over the same USB cable. Watching it is the fastest way to see WiFi/broker connection problems.

  • Easiest: in Chrome/Edge, the ESP Web Tools dialog has a “Logs & Console” option — open it to read the board live, right here.
  • Arduino IDE: Tools → Serial Monitor, set baud to 115200, pick the board's port (e.g. COM5).
  • You should see the === Tender Cells Starter Node === banner, the chip model, then WiFi and [MQTT] connected once it reaches your broker.
This browser can't flash directly. The one-click flasher needs Chrome or Edge on a laptop/desktop (it uses WebSerial). On phones, Safari, or Firefox it won't work — open tender-cells.web.app/flash in Chrome or Edge, or use the Learn mode Arduino IDE path.

Step 4 — Tell the coop your WiFi

  1. After flashing, the board makes its own WiFi network — TenderNode-Setup for the Starter Node, or ChickenTender-Setup for a coop.
  2. On your phone, join that network. A setup page pops up automatically.
  3. Pick your home WiFi (2.4 GHz) and type the password.
  4. Broker IP: leave it blank and the board auto-finds the npm run demo broker on your network (no IP typing). Or type the address the demo screen prints, e.g. 192.168.1.42, if your network blocks auto-discovery.
  5. Starter Node only: set Product type to which product this board stands in for — starter, chicken-tender, roaming-roost, duck-dock, bunny-burrow, goat-guardian, turkey-tower, pigeon-palace, watchtower, or custom to invent your own device.
  6. Build your own (optional): with custom, name a Species (e.g. axolotl) and a Threat label (e.g. hawk). Then the board's BOOT button fires that threat live into the dashboard — great for classrooms.
  7. Save. The board connects and shows up when you run tc status. Done! 🎉

Nothing secret is ever baked into the chip — you set the WiFi yourself, so it's safe to share boards. Teachers: see the Build Your Own Device classroom lesson.

What does "flashing" even mean? A microcontroller (the ESP32) is a tiny computer with no hard drive. "Flashing" copies a program into its memory so it runs that program every time it powers on. We're copying the Tender Cells coop program into your board.

What's on the chip after you flash

  • Bootloader — the first code that runs; it starts everything else.
  • Partition table — a map of what lives where in memory.
  • The coop program (firmware) — reads sensors, opens the door, runs the feeder, and listens for commands. This is the part you'd change to teach the coop new tricks.

How a command reaches the coop

your tap ─▶ app / tc CLI ─▶ REST ─▶ bridge ─▶ MQTT ─▶ ESP32 coop │ sensors / state ◀──────────────── MQTT ◀───────────── ESP32 publishes

The board talks MQTT — little JSON messages on named "topics" like tc/your-coop/cmd/door. Motion never goes through the cloud; it stays on your own network so it's fast and works without internet.

Three ways to put the program on, easiest first

  1. This page (Easy mode) — zero installs, just Chrome/Edge. Best for a first run or a classroom.
  2. Arduino IDE — needs installs: the Arduino IDE itself, plus the ESP32 board package (Boards Manager → "esp32"), plus the libraries (WiFiManager, PubSubClient, ArduinoJson v7). Then open firmware/starter-node (any product) or firmware/chicken-tender (full coop), pick your board + port, press Upload. Good when you want to read and change the code and watch the Serial Monitor.
    📖 Full step-by-step: Arduino IDE on-ramp guide (board settings, drivers, hardware needs, product types).
  3. PlatformIO — needs installs: Python + PlatformIO Core (or the VS Code extension). Then pio run -t upload from a terminal — it auto-fetches the toolchain and libraries. Same code, scripted builds, used by our automated tests.

All three flash the same program. Start at #1; graduate to #2 when you want to tinker.

Try changing something

Open firmware/chicken-tender/src/main.cpp. Find DOOR_OPEN_DEG near the top — that's how far the door swings. Change the number, flash again (Arduino IDE), and watch the door move differently. That's your first edit. 🛠️

Safety is baked in: the code refuses to move the arm while a chicken is detected, and an E-STOP command freezes every motor instantly. Read the comments to see how.
