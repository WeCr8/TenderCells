Step 2 — Plug the board into this computer

Use the USB cable. A light on the board should turn on.

Step 3 — Press the button

  1. A little window pops up asking which USB port. Pick the one that appears when the board is plugged in (often named CP210x, CH340, or USB Serial), then press Connect.
  2. Press Install Tender Cells — Chicken Tender and confirm. A green bar fills up.
  3. When it says done, the program is on the chip. Unplug or keep going to WiFi setup below. 🎉
❓ Board doesn't show up / install fails
  1. Swap the cable for a known data cable — #1 cause.
  2. Hold the BOOT button on the board, press the button above, keep holding BOOT until the green bar starts, then let go.
  3. Close any other program using the port (Arduino IDE Serial Monitor, etc.).
  4. Still stuck? Switch to Learn mode and use the Arduino IDE path.
This browser can't flash directly. The one-click flasher needs Chrome or Edge on a laptop/desktop (it uses WebSerial). On phones, Safari, or Firefox it won't work — open tender-cells.web.app/flash in Chrome or Edge, or use the Learn mode Arduino IDE path.

Step 4 — Tell the coop your WiFi

  1. After flashing, the board makes its own WiFi network called ChickenTender-Setup.
  2. On your phone, join that network. A setup page pops up automatically.
  3. Pick your home WiFi, type the password, and enter the broker IP (the address the npm run demo screen prints, e.g. 192.168.1.42).
  4. Save. The coop connects and shows up when you run tc status. Done! 🎉

Nothing secret is ever baked into the chip — you set the WiFi yourself, so it's safe to share boards.

What does "flashing" even mean? A microcontroller (the ESP32) is a tiny computer with no hard drive. "Flashing" copies a program into its memory so it runs that program every time it powers on. We're copying the Tender Cells coop program into your board.

What's on the chip after you flash

  • Bootloader — the first code that runs; it starts everything else.
  • Partition table — a map of what lives where in memory.
  • The coop program (firmware) — reads sensors, opens the door, runs the feeder, and listens for commands. This is the part you'd change to teach the coop new tricks.

How a command reaches the coop

your tap ─▶ app / tc CLI ─▶ REST ─▶ bridge ─▶ MQTT ─▶ ESP32 coop │ sensors / state ◀──────────────── MQTT ◀───────────── ESP32 publishes

The board talks MQTT — little JSON messages on named "topics" like tc/your-coop/cmd/door. Motion never goes through the cloud; it stays on your own network so it's fast and works without internet.

Three ways to put the program on, easiest first

  1. This page (Easy mode) — zero installs, just Chrome/Edge. Best for a first run or a classroom.
  2. Arduino IDE — install the IDE, add the ESP32 boards, open firmware/chicken-tender, pick your board + port, press Upload. Good when you want to read and change the code and watch the Serial Monitor.
  3. PlatformIO — for power users: pio run -t upload from a terminal. Same code, scripted builds, used by our automated tests.

All three flash the same program. Start at #1; graduate to #2 when you want to tinker.

Try changing something

Open firmware/chicken-tender/src/main.cpp. Find DOOR_OPEN_DEG near the top — that's how far the door swings. Change the number, flash again (Arduino IDE), and watch the door move differently. That's your first edit. 🛠️

Safety is baked in: the code refuses to move the arm while a chicken is detected, and an E-STOP command freezes every motor instantly. Read the comments to see how.
