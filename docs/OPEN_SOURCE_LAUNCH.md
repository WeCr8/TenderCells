# TenderCells Open Source Launch Plan

## Positioning

TenderCells is an open-source operating system for home farming automation and animal care. The project combines product registration, device telemetry, property simulation, firmware, hardware documentation, and robotics workflows so people can build their own systems or buy kits when they want a faster path.

## First 90 Days

### Month 1: Foundation

- Publish the GitHub repository with a clear README, contributing guide, and issue templates.
- Publish hardware documentation standards for every product family.
- Publish the first Chicken TenderCell build guide and garage test notes.
- Open good-first issues for documentation, mobile UI, product registry, and firmware testing.

### Month 2: Builder Content

- Post weekly build videos or photo logs.
- Invite homesteading, FFA, 4H, maker, and robotics communities to test docs.
- Add beta-builder intake issues and tag selected testers.
- Publish Roaming Roost and WatchTower AI simulation setup notes.

### Month 3: Community Proof

- Collect deployment photos, build logs, and lessons learned.
- Publish first case studies.
- Start monthly roadmap updates.
- Identify the first kit candidates: Coop Door Kit, Water Monitor, Feed Monitor, WatchTower AI.

## Channels

- GitHub: source, issues, discussions, releases, roadmap.
- YouTube: build logs, safety tests, prototype demos.
- Reddit: homesteading, backyard chickens, robotics, 3D printing, ESP32 communities.
- Facebook: homesteading, poultry, animal care, and maker groups.
- FFA and 4H: school-friendly build kits, STEM documentation, youth project guides.
- Discord: contributor support, beta-builder coordination, field-test reports.

## Success Metrics

- GitHub stars and forks.
- Active contributors.
- Open issues closed by community members.
- Discord members and useful field reports.
- Devices deployed or simulated.
- Beta-builder applications.
- Product documentation completeness by product family.

## Free and Paid Boundary

Free:

- Self-hosted dashboard.
- Open-source firmware.
- Product schemas, device registry, and simulation instructions.
- Build docs for community products.

Paid:

- Hardware kits.
- Complete systems.
- Cloud hosting.
- AI analytics.
- Consulting and custom integrations.

The paid path should save time without locking people out of self-hosting, repairing, remixing, or learning.
